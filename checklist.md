# Draft QA + Approval Checklist

## Safety
- Uses only facts in `truth_file.md`.
- No proprietary partner/vendor/customer details.
- No confidential metrics beyond approved outcomes.

## Clarity
- One core idea.
- Clear hook in first 2 lines.
- Ends with one explicit question.

## Format
- Long-form: <= 450 words.
- Short: <= 200 words.
- Closing line feels human and specific to the post (avoid canned CTA phrases like "Follow + Discuss.").

## Approval
- Mark approved in `approval_log.md`.
- Explicit publish confirmation required before posting.
