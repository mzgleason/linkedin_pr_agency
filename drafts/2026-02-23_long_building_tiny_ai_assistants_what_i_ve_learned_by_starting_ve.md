# Building tiny AI assistants: what I’ve learned by starting very small

This week I spent most of my tinkering time on three small projects: a Telegram-connected personal assistant on my laptop, an automated LinkedIn PR agent that drafts my weekly posts after a Friday lunch prompt, and a public directory prototype for local dance studios.

The practical lesson that kept surfacing: start tiny. The Telegram assistant began as a one-action MVP — create a ticket when I’m on a walk and don’t want to pause to open my laptop. That single, well-tested task made the rest of the work tractable. The LinkedIn PR agent follows the same pattern: a focused prompt → storyboard → draft post. Because each step has a clear test, I can iterate without accidentally breaking prior behavior.

I’m also reminded how seductive parallel projects can become once agents can push things forward for you. Agents reduce friction, which is powerful, but they also make it easier to accumulate half-finished ideas. The “last 20%” — polishing edge cases, reliable data ingestion, and human review workflows — still takes the most time.

These micro-automation patterns aren’t just academic. In product roles I’ve seen focused improvements compound: increased customer engagement by 40% through digital experience work, saved approximately $9M in annualized losses by resolving a routing flaw, and delivered $500K+ incremental value by integrating data science with better forms. Those outcomes come from lots of small wins, each delivered and validated.

My plan for next week: stabilize the Telegram assistant with a few more integration tests, push the LinkedIn PR agent to a “close to production” draft cadence, and scope the scraping requirements for the dance-studio directory so I can treat it like an explicit MVP.

What is one tiny automation you could build this week that would make tomorrow measurably easier?
