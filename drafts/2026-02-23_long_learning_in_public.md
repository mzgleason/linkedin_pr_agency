# 2026-02-23 (Long) - Why I built a "PR team" workflow for LinkedIn

I barely posted on LinkedIn before this.

Not because I had nothing to say, but because the effort always felt too large. A post wasn't just writing. It was framing, editing, second-guessing, and wondering if it was worth publishing.

Then AI made posting feel easy, but not always real. I could generate content fast, but it didn't always sound like me.

So I built a system in the middle: my input first, then a lightweight "PR team" workflow to refine it.

Here's the flow I'm using now:
- Intake (my raw idea, context, and point of view)
- Drafts (structured versions of the message)
- QA (clarity + audience value checks)
- Approval (final pass before posting)
- Publish (manual, intentional)

The key for me is this: AI helps with refinement, not authorship.

I still own the message. The workflow just removes friction and makes quality more consistent.

What changed immediately:
1. It feels smaller to ship one strong post at a time.
2. I spend less energy deciding and more energy improving.
3. My voice stays intact because the process starts with real personal input.

I'm sharing this as part of learning in public as a PM integrating AI into real workflows.

If you've ever felt like posting is "too much work" or AI outputs feel "not quite you," this hybrid model might help.
