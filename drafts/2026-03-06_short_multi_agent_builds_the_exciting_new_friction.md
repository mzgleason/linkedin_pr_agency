# Multi-agent builds: the exciting new friction

I’m wrestling with a new concept this week: multi-agent builds where several Codex sessions communicate with one another with specific task scopes. The pattern feels powerful for business-owner and broker workflows, but it introduces coordination and scope-tracking challenges that amplify my context-switching problem. Colleagues were energized by the Codex demo, which reassures me the direction is right — the work now is designing simple, observable contracts between agents so teams can trust the end-to-end behavior.
